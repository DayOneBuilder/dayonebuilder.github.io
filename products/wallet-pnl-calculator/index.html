Scope: rough wallet-level only. No tax lots, gas, staking accrual detail, or transfers between wallets you still control.

Formula: total P&L = withdrawals + current holdings value - deposits

Where this helps

You already pulled money out and want to know what the rest of the bag is doing

This separates capital already taken out from what is still sitting in the wallet so the open position is easier to judge.

You need a rough realized versus unrealized split before deeper cleanup

It gives you a fast estimate without forcing full spreadsheet reconstruction first.

You are pressure-testing a wallet summary that feels off

If deposits, withdrawals, and current value still do not reconcile in your head, this gives you one clean frame.

You want one plain-English readout instead of back-solving the wallet in notes

The output explains what the split means so you do not have to reconstruct the logic from memory.

Assumptions worth keeping in view

Deposits and withdrawals should be external flows

Do not count transfers between wallets you still control. That will distort both total and realized lines.

The current holdings cost basis is the key estimate

If that number is sloppy, the realized and unrealized split will be sloppy too. The total P&L line still stays simple.

Return percent only means something when deposits are above zero

If the wallet was funded in another way, the calculator leaves the percent line out instead of inventing one.

This is wallet-level math, not tax-lot accounting

Use it as a sanity check before deeper spreadsheet or tax software work, not as a replacement for lot tracking.

This is a rough decision aid, not tax or legal advice. If it saves you from trusting a bad wallet number, you can support future updates here. If not, keep using it.
