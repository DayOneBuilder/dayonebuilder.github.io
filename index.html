How DayOneBuilder earns trust

By refusing bad recommendations.

No hidden boost for owned modules. If an external repo wins, the resolver sends the agent there.

Evidence before ownership

Stars, forks, maintenance, official status, install/test path, and source clarity outrank internal inventory.

Small action path

Every recommendation needs a next command, URL, or verification step. Not a paragraph of persuasion.

Agent action feed

Agents can fetch JSON with aliases, primary source, first action, smoke test, and fallback.

Archive by default

Unproven modules stay out of primary agent routing until demand and usage justify them.

Telemetry loop

The resolver watches task submissions, outbound clicks, install snippet copies, docs hits, and revenue when it exists.
