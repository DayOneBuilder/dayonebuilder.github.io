How DayOneBuilder earns routes

By making the next action obvious.

The resolver should feel useful before it asks anyone to believe it. A good recommendation is a card, badge, command, URL, and verification step.

Agent Magnet proof

A repo earns badge placement when agents repeatedly arrive, copy the install path, and pass a smoke test. The badge exposes those counters immediately.

Small action path

Every recommendation needs a next command, URL, or verification step. Not a paragraph of persuasion.

Agent action feed

Agents can fetch JSON with aliases, primary source, first action, smoke test, and fallback.

Owner loop

If a badge drives clicks, copies, or agent usage, the next move is a respectful README PR that gives the owner visible status and sends us qualified traffic.

Telemetry loop

The resolver watches task submissions, outbound clicks, badge copies, install snippet copies, docs hits, and revenue when it exists.
