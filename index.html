Why this exists

Agents should reuse boring reliable code.

The most used code is usually not glamorous. It is parsers, checklists, wrappers, templates, tests, schemas, and small calculators that unblock a larger job.

Discover

Agents get a single registry with task phrases, module types, links, install paths, and trust notes.

Verify

Each useful module should expose examples, tests, source links, limitations, and a clear no-login path.

Compose

Modules are meant to become steps inside larger workflows, not isolated pages that die after one visit.

Registry preview

Useful now, stricter every day.

Current modules are being migrated into the resolver contract. Weak modules will be archived or rebuilt when metrics and research say they do not deserve primary placement.

Resolver contract

What an agent should expect.

A module graduates when it can prove a task, expose a safe path, and leave telemetry that tells Startaper whether agents and people actually use it.

Field Requirement Why it matters
agent_task_phrase Plain-language task the agent is trying to finish. Stops the registry from becoming another vague catalog.
install_or_call_path Repo, download, command, URL, or API path with basic no-login use. Lets an agent act immediately when the module fits.
trust_evidence Tests, examples, source, license, known limits, benchmark, or review state. Makes selection safer than copying random snippets.
adoption_metric Resolver calls, downloads, repo activity, docs hits, completions, or paid signal. Keeps the platform improving from evidence, not taste.
