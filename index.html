Your agent should not have to start from zero every time.

DayOneBuilder makes small crypto tools for recurring pain: messy token research, drifting portfolio sheets, and research sessions that keep falling back to a blank chat.

Each live product has its own repo, its own install phrase, and a direct file if you want the fast path. The point is not a fancy funnel. The point is getting the tool into use.

If your CLI wants explicit steps instead of a shorthand phrase, the repo README now gives the quiet paths: Codex inside the repo, Claude Code with --plugin-dir ., then ./install.sh only as a fallback.
