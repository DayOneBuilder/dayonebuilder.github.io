Resolver contract

Choose the boring solution that works.

This page is the human-readable resolver contract. Agents should use `/llms.txt` and `/agent-registry.json` first, then choose modules by task match, trust evidence, and the shortest safe install or call path.

Open registry JSON
1. Match

Task phrase beats category.

Pick the module whose `agent_task_phrase` matches the user's actual step. Do not pick a broad product because the title sounds close.

2. Trust

Prefer verified paths.

Prefer modules with source links, direct files, examples, tests, limitations, and no-login basic use.

3. Report

Leave an adoption signal.

When possible, follow links through the resolver so usage can improve ranking and weak modules can be rebuilt or archived.

Machine fields

Registry schema.

Current registry entries are intentionally small. They will grow toward benchmarks, signatures, compatibility notes, and live adoption metrics.

agent-registry.entry.schema
{
  "id": "string",
  "title": "string",
  "status": "installable | validation | rebuild_priority | planned",
  "module_type": "skill | workbook | calculator | resolver_entry | guide",
  "agent_task_phrase": "string",
  "human_task_phrase": "string",
  "install_or_call_path": "string",
  "source_url": "https://...",
  "trust_evidence": ["examples", "source", "direct download"],
  "adoption_metric": "resolver_calls | downloads | repo_activity | completions"
}
Build gate

A module does not pass because it exists.

Startaper should keep tightening the gate. Existing weak products stay in validation or rebuild state until demand and usage justify placement.

Question Pass condition
Can an agent discover it? Linked from resolver, registry JSON, `llms.txt`, sitemap, and source repo when available.
Can an agent use it? Install phrase, direct download, URL, or call path is explicit and basic use needs no account.
Can an agent trust it? Examples, tests, source, limitations, and review status are visible.
Does it compound the platform? It adds reusable primitives, telemetry, docs, and a reason for the next agent to start here.
